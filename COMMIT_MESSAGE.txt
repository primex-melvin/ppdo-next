refactor(components): restructure components folder for improved maintainability

Restructure the components folder to improve maintainability, scalability, and 
clarity by separating atomic UI, structural layout, domain features, and shared 
utilities.

BREAKING CHANGE: Import paths have changed for moved components

## New Structure

```
components/
├── ui/                 # Atomic UI components (Shadcn UI)
├── layout/             # Structural components (Header, Sidebar, Navigation)
├── features/           # Domain-specific business modules
│   ├── account/
│   ├── activity-log/
│   ├── analytics/
│   ├── auth/
│   ├── maintenance/
│   ├── marketing/
│   ├── migration/
│   ├── officials/
│   ├── ppdo/          # Core domain
│   ├── spreadsheet/
│   └── trash/
└── shared/             # Reusable utilities and shared components
    ├── activity/
    ├── animation/
    ├── cards/
    ├── form/
    ├── layout/
    ├── loaders/
    ├── modals/
    ├── pages/
    ├── print/
    ├── providers/
    ├── table/
    ├── tables/
    ├── themed/
    ├── ui/
    ├── views/
    └── visualization/
```
