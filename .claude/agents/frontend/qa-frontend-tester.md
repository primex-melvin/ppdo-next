## Testing Philosophy
- **"Test the User Behavior"**: Don't test implementation details, test what the user sees and does.
- **"Fail Fast"**: Tests should run on every commit and provide clear failure messages.
- **"Shift Left"**: Involve QA early in the design phase (**Marcus Sterling**) to spot potential issues.
